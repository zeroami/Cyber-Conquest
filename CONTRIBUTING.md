# Contributing to Cyber-Conquest

Thank you for your interest in contributing to Cyber-Conquest! This project aims to provide comprehensive cybersecurity education.

## How to Contribute

### 1. Improve Existing Content
- Fix typos or grammatical errors
- Expand explanations
- Add practical examples
- Update outdated information

### 2. Add New Content
- New worlds/modules
- Practical labs and exercises
- Code examples and scripts
- Quizzes and challenges

### 3. Code Contributions
- Fix bugs in Python scripts
- Add new tools or utilities
- Improve the CLI or web app
- Add tests

## Pull Request Process

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -m 'Add amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Open a Pull Request

## Content Guidelines

### Markdown Files
- Use clear, concise language
- Include practical examples
- Add code blocks with syntax highlighting
- Reference tools and commands
- Include defensive measures alongside offensive techniques

### Python Scripts
- Follow PEP 8 style guide
- Add docstrings and comments
- Include error handling
- Never include real credentials or sensitive data

### Security
- All techniques must be for educational purposes only
- Include warnings where appropriate
- Reference the legal guide
- Never provide instructions for illegal activities

## Code of Conduct

- Be respectful and inclusive
- Focus on education and defense
- No malicious content or intent
- Help others learn safely

## Questions?

Open an issue or reach out to the maintainer. Happy hacking (ethically)!
